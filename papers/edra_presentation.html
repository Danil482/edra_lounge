08a PREREQUISITE 1

Do Natural Groups Exist?

Before testing EDRA, verify the audience has exploitable structure.

  • Are there natural groups in the recipient population?
  • Silhouette > 0.3 → clusters are real, not noise
  • χ² p < 0.05 → reply rates differ between clusters
  • If this fails → audience is homogeneous, no need for EDRA
GOOD
3–5 clusters, distinct reply rates → proceed to Level 2
BAD
One mega-cluster or p > 0.05 → features too sparse, need richer profiles

08b PREREQUISITE 2

Does Optimal Strategy Differ by Cluster?

Personalization is known to help. The question: does it help enough on 804 noisy rows to justify the machinery?

  • πedra vs πbest_single — not “personalization vs nothing”
  • With 804 noisy rows, cluster-conditional may lose to the best global strategy
  • Doubly robust estimator corrects for campaign assignment bias (Dudík 2011)
GOOD
πedra > πbest_single → personalization works, clusters are load-bearing
BAD
πbest_single ≈ πedra → one strategy fits all, clustering is unnecessary overhead
08c THE TEST

Can EDRA Learn What Works?

The actual contribution: automatic discovery of cluster-conditional strategies + online revision when they decay.

  • Prequential: predict first, learn after — no lookahead bias
  • EDRA revises structured rules; LinUCB updates weight vectors
  • What this catches: overfitting to early data, drift, flat generic rules that never specialize
GOOD
V̂DR rises over batches, EDRA outperforms LinUCB → structured rules + revision > weight vectors
BAD
Flat curve or EDRA ≤ LinUCB → the rule induction/revision machinery adds no value over simpler bandits

10 LANDSCAPE

Different Problems, Different Layers

SOInspire ExpeL LinUCB EDRA
Layer Content generation Knowledge extraction Action selection Strategy adaptation
Optimizes Text quality (CTR) Task accuracy Expected reward Reply rate per cluster
Personalizes for Author (style transfer) — User context Recipient cluster
Clusters Content styles → personas — — People → audience segments
Feedback signal CTR / engagement None (fixed) Reward (numeric) Reply / no-reply → rule revision
Output Ready-to-post text Text insight list Arm index 5-slot rule + generated message

SOInspire (Farseev et al., SIGIR ’26) solves what to say — content quality. EDRA solves whom to say what — strategy selection per audience segment. Complementary layers, not competing systems.

11 CONTRIBUTIONS

What EDRA Demonstrates

  • Cluster-conditional rules — not flat insights
